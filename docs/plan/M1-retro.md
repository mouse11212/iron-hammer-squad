# M1 复盘（fincards-m1-mutation-gate）

> 日期 2026-06-17。M1 = 变异测试确定性硬门(第一条有据可依的 harness 约束)。对应 V4 §4.6、§4.1、§2。

## 交付结果

- fincards 接入 StrykerJS 变异门;只变异纯逻辑 parse/filterToday/render。
- **变异分数 67.21%(M0 基线) → 100%**(59/59 非等价变异被杀);2 个等价变异在源码内显式豁免(带理由)。
- 设 break 阈值 90 的硬门;门有效性自检通过(未测文件纳入→分数跌破阈值→退出码 1)。
- 单测 13 → **24**;快 gate 仍全绿。

## M1 验证的命题(KB 痛点二)

M0 "测试全绿" ≠ "测试够强"——变异测试当场揭穿:render 仅 56.52%、parse 66.67%。补强后达 100%。证明"绿"需要被审计,变异测试是把"判卷人"换成不会偷懒的机器(确定性 sensor)。

## 暴露的、可固化的点(→ M2 候选)

1. **测试强度需常态化**:变异门是合并前慢 sensor,M0 当时缺它。→ M2/集成阶段应把变异门接入 pre_merge / CI(V4 §9 军规8)。
2. **等价变异判定靠人**:本轮 2 个等价变异由人工判断+豁免。→ 多角色编排(M2)中,这类判定可作为评审 Agent 的一项确定性输入。
3. **gate 分层已成形**:快 gate(每次) vs 变异门(合并前)的时序分层,是 §4.1 的实际落地,M2 编排时应让不同角色触发不同层。
